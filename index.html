프로젝트 소개

크라우드 펀딩 플랫폼인 Wadiz를 참고하여 제작하였습니다.

저희가 제작한 플랫폼에서는 -
1. 아이디어를 등록하여 다른 사용자로부터 자금을 후원 받습니다.
2. 후원금이 목표금액에 도달하면 리워드라는 이름의 보상을 후원자에게 제공합니다.
3. 후원금이 달성된 경우 플랫폼 내 스토어에 등록하여 판매까지 이어질 수 있습니다.

개발환경 및 사용기술

1. 웹애플리케이션 서버는 톰캣을 사용하였습니다.
2. 스프링을 사용하여 MVC패턴으로 컨트롤러, 서비스 단을 분리하여 구현하였고 뷰는 JSP를 사용하였습니다.
3. 웹페이지 제작에는 자바스크립트, CSS를 기본적으로 사용하고 제이쿼리, 부트스트랩, Vue.js도 사용하였습니다.
4. 데이터베이스 오라클(11g 버전)을 사용하였고 E-R diagram 제작에는 eXERD를 사용하였습니다.
5. 마이바티스를 사용하여 매퍼 내에 쿼리문을 작성하여 데이터베이스의 정보를 가져왔습니다.

담당 파트

1. 메인 페이지
2. 로그인 페이지
3. 회원가입 페이지
4. 아이디/비밀번호 찾기 페이지

구현 내용


1. 메인 페이지

1. 스크롤링을 통한 페이징을 구현

1-1. AJAX와 Vue.js를 활용하여 화면의 스크롤을 끝까지 내렸을 때 리스트가 출력되도록 구현하였습니다. 한페이지 당 15개의 아이템을 가져오며 최대 3페이지까지 불러옵니다.
1-2. javascript의 InterceptionObserver 객체를 사용하여 이미지가 화면에 보이는 순서대로 로딩되도록 구현하였습니다.

1-3. 로그인 상태인 경우는 사용자의 관심 카테고리 정보를 사용하여 해당 카테고리에 속하는 리스트만을 출력하도록 구현하였습니다.

2. AJAX 및 소켓통신을 통한 알림기능 구현

2-1. 페이지 로드 이후 초기 알림 정보를 수신할 때는 세션에 저장된 유저 정보를 이용하여 AJAX를 통해 업데이트하도록 구현하였습니다.
2-2. 알림 정보는 내가 문의한 내용에 답변이 달린 경우 업데이트 되도록 구현하였습니다.
2-3. 해당 알림을 클릭하면 내가 문의한 내용으로 이동하도록 구현하였습니다.

2-4. 소켓 통신을 이용하여 페이지를 이동하지 않더라도 알림정보를 수신할 수 있도록 구현하였습니다.
2-5. 문의한 내용에 답변이 달린 경우 상단에 메시지가 팝업되며 알림의 개수가 +1 됩니다.

2. 로그인 페이지

1. 로그인 실패

1-1. 아이디를 입력하지 않은 경우 아이디를 입력해달라는 알림이 표시되도록 구현하였습니다.
1-2. 비밀번호를 입력하지 않은 경우 비밀번호를 입력해달라는 알림 표시되도록 구현하였습니다.
1-3. 아이디/비밀번호를 입력했으나 올바르지 않은 경우 올바른 아이디/비밀번호를 입력해달라는 알림이 표시되도록 구현하였습니다.
1-4. 올바른 아이디/비밀번호인지는 AJAX를 통해 서버와 통신하여 확인하도록 구현하였습니다.

1-5. 올바른/아이디 비밀번호를 입력했으나 해당 아이디가 정지상태인 경우 정지된 계정이라는 알림이 표시되도록 구현하였습니다.
1-6. 계정정지는 데이터베이스 내 회원 테이블 구현 시 활성상태를 나타내는 컬럼을 Date형식으로 추가하여 구현하였습니다. Date가 현재 시간보다 미래인 경우 로그인이 되지 않도록하여 회원 정지, 회원 탈퇴 기능을 구현하였습니다.

2. 로그인 성공

2-1. 로그인 이후 유저 정보를 세션에 저장하고 메인 페이지로 리다이렉트 하도록 구현하였습니다.

2-2. 만약 로그인을 요구하는 페이지로부터 로그인 페이지로 이동된 경우 로그인 이후 해당 페이지로 다시 이동하도록 구현하였습니다.

3. 회원가입 페이지

1. 이메일 검증

1-1. 정규표현식을 사용하여 입력값이 이메일 형식이 아닌 경우 알림이 표시되도록 구현하였습니다.
1-2. 이메일 형식의 입력을 하였더라도 이미 등록된 이메일인 경우 알림이 표시되도록 구현하였습니다.
1-3. 이미 등록된 이메일인지 확인하는 것은 해당 입력 칸을 빠져나온 순간 AJAX로 서버와 통신하여 등록여부를 확인하는 방식으로 구현하였습니다.

1-4. 올바른 이메일 형식이고, 등록되지 않은 이메일인 경우 사용가능한 이메일이라는 알림이 표시되도록 구현하였습니다.

2. 비밀번호 검증

2-1. 정규표현식을 사용하여 입력한 비밀번호가 숫자, 문자, 특수문자의 조합이며 8~20자 사이가 되도록 구현하였습니다.

2-2. 비밀번호확인의 경우 앞서 입력한 비밀번호가 같은지 검증하도록 구현하였습니다.

3. 이메일 인증

3-1. 모든 입력을 완료하고 회원가입을 누른 경우 해당 이메일로 랜덤한 1~9999 사이의 숫자의 인증번호를 발송하도록 구현하였습니다.
3-2. 이메일 발송을 위해 JAVA의 Mail라이브러리를 사용하였습니다.
3-3. 동시에 여러명이 회원가입을 하는 경우도 대응할 수 있도록 메일 발송시 해당 인증번호를 인증을 요청한 이메일 정보와 함께 데이터 베이스에 저장하도록 구현하였습니다.
3-4. 해당 테이블에는 Used 칼럼을 추가하여 한번 사용된 인증번호의 경우 재사용이 되지 않도록 구현하였습니다.
3-5. 3분의 제한시간을 두어 해당 시간이 지난 경우 서버에서 인증번호 테이블에 접근하여 Used 칼럼을 업데이트하여 인증번호가 더 이상 사용될 수 없도록 구현하였습니다.
3-6. 올바른 인증번호를 입력한 경우 회원가입이 완료되고 로그인 페이지로 이동하도록 구현하였습니다.

4. 아이디/비밀번호 찾기 페이지

1. 아이디/비밀번호 찾기 전환

1-1. 아이디와 비밀번호 찾기는 비슷한 기능이기 때문에 페이지 이동 없이 한 화면에 구현하였습니다.
1-2. 구현을 위해 Vue.js를 사용하였으며 아이디찾기와 비밀번호찾기를 각각 컴포넌트화하여 사용하였습니다.

2. 아이디찾기

2-1. 해당 플랫폼에서는 이메일을 아이디로 사용하기 때문에 이메일을 입력하여 가입여부를 확인하도록 구현하였습니다.
2-2. 가입여부 확인을 클릭하는 순간 서버와 AJAX로 통신하여 가입여부를 확인하여 가입된 경우 이미 가입되었다고 알림이 표시되도록 구현하였습니다.

3. 비밀번호찾기

3-1. 비밀번호를 재설정하는 방법으로 비밀번호 찾기 기능을 수행하도록 구현하였습니다.
3-2. 등록되지 않은 이메일을 입력한 경우 등록되지 않았다는 알림이 표시되고, 등록된 경우 변경할 비밀번호를 입력할 수 있는 입력창이 나타나도록 구현하였습니다.

3-3. 비밀번호를 재설정하는 경우도 회원가입과 마찬가지로 8~20자의 문자, 숫자, 특수문자 조합의 비밀번호를 요구하도록 구현하였습니다.
3-4. 동일한 비밀번호를 한번 더 입력하여 비밀번호 확인 작업을 마친 경우 변경완료 버튼이 나타나도록 구현하였습니다.

3-5. 변경 완료를 클릭하는 경우 회원가입과 마찬가지로 이메일 인증을 요구하도록 구현하였습니다.
3-4. 이메일 인증과 마찬가지로 JAVA Mail라이브러리를 사용하여 메일을 발송하며, 랜덤한 숫자를 생성하고 데이터베이스 인증 테이블에 해당 인증번호를 추가하여 인증을 구현하였습니다.
3-5. 인증번호가 일치하는 경우 변경이 완료되며, 로그인 페이지로 이동하도록 구현하였습니다.
